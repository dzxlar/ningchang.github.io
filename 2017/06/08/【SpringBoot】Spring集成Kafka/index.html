application.yml配置

  • group-id: 组别id，确保topic分区正确分配给消费者
  • auto-offset-reset: 确保消费者组接收的消息是最新发送的

Kafka Spring Producer

KafkaTemplate

KafkaTemplate封装了Producer一系列便利方法发送数据到topic中。同时提供同步和异步方法返回一个Future对象

  • sendDefault 提供了一个默认主题模板
  • excute方法直接方法Kafka底层的Producer
  • metric/partitionsFor 委托Kafka底层的方法

如何使用模板。需要配置生产者工厂和kafka模板构造函数

使用Message< ? >,topic,partition和key这些Spring提供了一些关于Kafka的消息头：

  • KafkaHeaders.TOPIC
  • KafkaHeaders.PARTITION_ID
  • KafkaHeaders.MESSAGE_KEY
  • KafkaHeaders.TIMESTAMP

@Configuration
@EnableKafka
public class KafkaConfig {

@Bean
KafkaListenerContainerFactory<ConcurrentMessageListenerContainer<Integer, String>>
                    kafkaListenerContainerFactory() {
    ConcurrentKafkaListenerContainerFactory<Integer, String> factory =
                            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory());
    factory.setConcurrency(3);
    factory.getContainerProperties().setPollTimeout(3000);
    return factory;
}

@Bean
public ConsumerFactory<Integer, String> consumerFactory() {
    return new DefaultKafkaConsumerFactory<>(consumerConfigs());
}

@Bean
public Map<String, Object> consumerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, embeddedKafka.getBrokersAsString());
    return props;
}

}

Container Thread Naming

监听器容器有2个执行任务(一个调用消费者，一个调用监听者)

设置 enable.auto.commit = false ，通过设置ContainerProperties的consumerExecutor and listenerExecutor2个属性自定custom executors。如果使用pooled executors确保有足够的线程处理并发性

ConcurrentMessageListenerContainer用于并发处理
