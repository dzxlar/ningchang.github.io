Hive学习笔记（一）

概念

Hive是一个基于Apache Hadoop的数据仓库。对于数据存储与处理，Hadoop提供了主要的扩展和容错能力。

对于大量的数据，使得数据汇总，查询和分析更加简单。它提供了SQL，允许用户更加简单地进行查询，汇总和数据分析。

同时，Hive的SQL给予了用户多种方式来集成自己的功能，然后做定制化的查询，例如用户自定义函数

结构

  • 数据库
    • 相当于命名空间，避免表，视图，分区，列等等的命名冲突
  • 表
    • 相同数据库的同类数据单元
  • 分区
    • 每个表可以有一个或多个用于决定数据如何存储的分区键
  • 桶
    • 每个分区的数据，基于表的一些列的哈希函数值，又被分割成桶

数据类型

  • Integer
    • tinyint、smallint、int、bigint
  • 布尔
    • boolean
  • 浮点数
    • float
    • double
  • 定点数
    • decimal
  • 字符串
    • string
    • varchar
    • char
  • 日期和时间
    • timestemp
    • date
  • 二进制
    • binary
  • 复杂类型
    • 结构体类型(Stuct)
      • 列a{b int,c,int},通过列.属性访问(例如：a.b,a.c）
    • Map键值对类型
      • 通过[‘colName’]访问
    • 数组
      • 通过[‘n’]访问数组

内置运算和函数

线程池

合理运用线程池的优点

  1. 降低资源消耗
  2. 提高响应速度
  3. 提高线程的可管理性

线程池步骤

  1. 提交任务
  2. 核心线程池未满、创建线程执行任务
  3. 核心线程池已满、队列未满，存储在队列中
  4. 队列已满、创建线程执行任务

创建线程池

public ThreadPoolExecutor(int corePoolSize,
                          int maximumPoolSize,
                          long keepAliveTime,
                          TimeUnit unit,
                          BlockingQueue<Runnable> workQueue)

向线程池提交任务

  • execute()提交不需要返回值的任务

    threadPool.execute(new Runnable());

  • submit()提交包含返回值的任务（返回Future对象，get()方法取得返回值）

    Future<Object> task = executor.submit(returnValueTask);
task.get();

关闭线程池

遍历线程池中的工作线程，然后逐个调用interrupt方法中断线程。可能无法响应中断的线程无法进行

  • shutdown()
  • shutdownNow()

shutdownNow除了关闭线程以外还返回正在等待执行任务线程的列表。而shutdown只是关闭

【多线程】并发工具类

发表于 2017-06-26 | 分类于 多线程

CountDownLatch

允许一个或多个线程等待其他线程完成操作

具体实例

解析Excel中的每个Sheet数据。考虑通过多线程解析每个Sheet数据。所有线程解析完毕后，才算解析完成

方法一: join()

Thread parse1 = new Thread(new Runnable() {

            @Override
            public void run() {
                System.out.println(1);
            }
        });
        Thread parse2 = new Thread(new Runnable() {

            @Override
            public void run() {
                System.out.println(2);
            }
        });
        parse1.start();
        parse2.start();
        parse1.join();
        parse2.join();
        System.out.println("all");

join用于当前执行线程等待join线程执行结束。原理：不停检查join线程是否存活。如果join线程存活则让当前线程永远等待.wait(0)会让线程永远等待。直到join线程中止后，线程的notifyall()方法被调用

方法二: CountDownLatch

final int N = 2;
CountDownLatch latch = new CountDownLatch(N);
new Thread(new Runnable() {

     @Override
     public void run() {
         System.out.println(1);
         latch.countDown();
         System.out.println(2);
         latch.countDown();
     }
 }).start();
 latch.await();
 System.out.println(3);

CountDownLatch传入int类型的作为计数器。等待N个点完成

每次调用countDown()方法时N就会减少1.await()会阻塞当前线程，直到N变成0

countDown可以用在任何地方。所以可以代表N个线程，也可以代表1个线程内执行的N个步骤

如果其中某个解析sheet线程处理较慢。不让主线程一直等待，可以使用带时间的await()的方法。等待特定时间后就会不在阻塞当前线程

注意：1. 计数器必须大于等于0。 2. 当N=0时，await()不会阻塞当前线程。3.CountDownLatch不能重新初始化或者修改内部计数器的值。 3.一个线程调用countDown方法happens-before，另外一个线程调用await

Cyclicbar

让一组线程到达一个屏障（同步点）被阻塞。直到最后一个线程达到屏障时，线程放行，所有被屏障拦截的线程才会继续执行

final int N = 2;
CyclicBarrier barrier = new CyclicBarrier(N);
new Thread(new Runnable() {
     @Override
     public void run() {
         try {
             barrier.await();
         } catch (InterruptedException e) {
             e.printStackTrace();
         } catch (BrokenBarrierException e) {
             e.printStackTrace();
         }
         System.out.println(1);
     }
 }).start();
 try {
     barrier.await();
 } catch (BrokenBarrierException e) {
     e.printStackTrace();
 }
 System.out.println(2);

由于主线程和子线程的调度通过cpu决定执行的。所以先后顺序可能是1,2或者是2,1

如果new CyclicBarrier(3)则线程会永远等待。因为没有第三个线程执行await()方法。所以会一直等待。

CyclicBarrier(int parties,Runnable barrierAction),用于在线程达到屏障时候，优先执行barrierAction。处理更复杂的业务场景

    final int N = 2;
       CyclicBarrier barrier = new CyclicBarrier(N,new ThreadA());
       new Thread(new Runnable() {
            @Override
            public void run() {
                try {
                    barrier.await();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } catch (BrokenBarrierException e) {
                    e.printStackTrace();
                }
                System.out.println(1);
            }
        }).start();
        try {
            barrier.await();
        } catch (BrokenBarrierException e) {
            e.printStackTrace();
        }
        System.out.println(2);;                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      
}

static class ThreadA implements Runnable{

    @Override
    public void run() {
        System.out.println(3);
    }

}

CountDownLatch和CyclicBarrier的区别

1、 CountDownLatch计数器只使用1次，而CyclicBarrier的计数器可以通过reset()重置

  1. CyclicBarrier可以获得阻塞线程数量。isBroken()判断是否中断线程

Semaphore

控制同时访问资源的线程流量（限流）。相当于每次只允许特定数量的人进入，再根据里面出来多少的数量决定再次进入多少，以此类推。

final int N = 10;
Semaphore semaphore = new Semaphore(N);
ExecutorService threadPool = Executors.newFixedThreadPool(30); 
for (int i = 0; i < 30; i++) {
    threadPool.execute(new Runnable() {
     @Override
     public void run() {
         try {
             semaphore.acquire();
             System.out.println("save data");
         } catch (InterruptedException e) {
             e.printStackTrace();
         }
         semaphore.release();
     }
 });
}
threadPool.shutdown();

【多线程】线程中的原子操作基本概念

发表于 2017-06-23 | 分类于 多线程 ， Atomic

基本概念

当一个请求在多线程环境下涉及到更新变量并且是原子性增长的，在线程非安全情况下可能会获得不同的值（例如：用户的点赞 A用户（A线程）点赞+1，B用户（B线程）点赞+1，最终得到的结果可能是2，不是3)

为了保证程序在多线程环境下计数正确。Java引入atomic包（包内的原子操作类为更新一个变量提供一种高效、简单、线程安全的方式）

因为变量类型有很多种所以在Atomic包内提供了13种原子操作类;属于4种类型的原子更新方式。分别是原子更新基本类型、原子更新数组、原子更新引用、原子更新属性(字段)

Atomic包里的类基本都是使用Unsafe实现的包装类

原子更新基本类

  • AtomicInteger:原子更新整型
  • AtomicBoolean:原子更新布尔类型
  • AtomicLong:原子更新长整型

原子更新数组

  • AtomicIntegerArray 原子更新整型数组的元素
  • AtomicLongArray 原子更新长整型数组的元素
  • AtomicReferenceArray 原子更新引用类型数组的元素

原子更新引用类型

  • AtomicReference 更新引用类型
  • AtomicReferenceFieldUpdater 更新引用类型的字段
  • AtomicMarkableReference 更新带有标记位的引用类型

【源代码】ConcurrentHashMap

发表于 2017-06-22 | 分类于 Java

ConcurrentHashMap出现的原因

  1. HashMap线程不安全。在多线程环境下容易出现死循环（环状结构数据）
  2. HashTable通过synchronized关键字保证线程安全。在锁竞争激烈的环境下效率低下

ConcurrentHashMap

采取锁分段技术：每把锁只锁住其中一部分数据。这样多线程环境下访问容器里不同数据段的数据不会产生锁竞争。

ConcurrentHashMap类结构

  1. ConcurrentHashMap< K,V > 继承 AbstractMap< K,V >
  2. ConcurrentHashMap由Segment和HashEntry数组结构组成
    • Segment是重入锁：扮演锁的角色
    • HashEntry存储键值对数据

Segment和HashMap类似是数组和链表结构
一个Segment包含一个HashEntry数组。每个HashEntry是一个链表结构的元素。每个Segment保护HashEntry的每个元素（访问之前都会获得锁）

ConcurrentHashMap的初始化

ConcurrentHashMap的3个初始化参数int initialCapacity, float loadFactor,int concurrencyLevel初始化Segment数组、SegmentShift段偏移量、SegmentMask段掩码和每个Segment的HashEntry实现的

ConcurrentHashMap的操作

  • get
    • get方法高效的原因：是不需要加锁。只有在值为空时才加锁重读
  • put
  • size

Hadoop学习笔记 —— 单节点安装(伪分布式和本地安装)

Hadoop概念

是一款可靠的、可扩展、分布式计算的开源的软件

Hadoop主要模块

  1. Hadoop Common(公共模块):支持其他Hadoop模块的常见运用程序
  2. Hadoop Distributed File System(HDFS)：分布式文件系统
  3. Hadoop YARN：统一管理调度和资源管理的框架
  4. Hadoop MapReduce：YARN-based系统并行处理大型数据集

Hadoop其他常用模块(列举部分)

  1. HBase
  2. Hive
  3. Pig
  4. Spark
  5. Zookeeper

Hadoop下载

Hadoop Download

Hadoop 单节点安装

centos添加新用户授权

Hadoop安装前提

  1. Java环境(Java7或者Java8)
    jdk下载
  2. ssh / pdsh
    centos6安装ssh

Hadoop启动集群

  1. 解压hadoop.tar.gz

  2. hadoop配置java环境

    vi  etc/hadoop/hadoop-env.sh

# set to the root of your Java installation
export JAVA_HOME=/usr/java

  3. 启动文档脚本

    bin/hadoop

Hadoop集群支持3种模式

  1. 本地模式
  2. 伪分布式模式
  3. 分布式集群模式

本地模式

为了方便调试，hadoop默认是一个单一节点(non-distributed)模式。

  1. 复制hadoop配置文件作为输入流

    hadoop> mkdir input
hadoop> cp etc/hadoop/*.xml input

  2. 输出写入到输入模式

    hadoop> bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-x.x.x.jar grep input output 'dfs[a-z.]+'
hadoop> cat output/*

伪分布式

etc/hadoop/core-site.xml

<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>

etc/hadoop/hdfs-site.xml

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>

设置ssh不需要密码

  1. 首先判断localhost是否需要密码

    ssh localhost

  2. 如果需要密码输入以下命令

    ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
  cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
  chmod 0600 ~/.ssh/authorized_keys

运行

本地运行

  1. 格式化文件系统

    bin/hdfs namenode -format

  2. 设置NameNode为守护进程

    sbin/start-dfs.sh

    hadoop守护进程日志目录($HADOOP_LOG_DIR目录 (默认为$HADOOP_HOME/logs)

  3. 访问NameNode（默认情况是可访问的)

    • NameNode - http://localhost:9870/

  1. 使所需的HDFS目录执行MapReduce工作

    bin/hdfs dfs -mkdir /user
  bin/hdfs dfs -mkdir /user/<username>

  2. 将输入文件复制到分布式系统

    bin/hdfs dfs -mkdir input
  bin/hdfs dfs -put etc/hadoop/*.xml input

  3. 运行实例

    bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-x.x.x.jar grep input output 'dfs[a-z.]+'

  1. 检查输出文件

    7.1. 将分布式文件系统的输出文件复制到本地文件系统

    bin/hdfs dfs -get output output
  cat output/*

    7.2. 分布式系统查看输出文件

    bin/hdfs dfs -cat output/*

  2. 停止守护进程

    sbin/stop-dfs.sh

YARN运行

etc/hadoop/mapred-site.xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
</configuration>
etc/hadoop/yarn-site.xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.env-whitelist</name>
        <value>JAVA_HOME,HADOOP_COMMON_HOME,HADOOP_HDFS_HOME,HADOOP_CONF_DIR,CLASSPATH_PREPEND_DISTCACHE,HADOOP_YARN_HOME,HADOOP_MAPRED_HOME</value>
    </property>
</configuration>

  • auto-offset-reset: 确保消费者组接收的消息是最新发送的

Kafka Spring Producer

KafkaTemplate

KafkaTemplate封装了Producer一系列便利方法发送数据到topic中。同时提供同步和异步方法返回一个Future对象

  • sendDefault 提供了一个默认主题模板
  • excute方法直接方法Kafka底层的Producer
  • metric/partitionsFor 委托Kafka底层的方法

如何使用模板。需要配置生产者工厂和kafka模板构造函数

使用Message< ? >,topic,partition和key这些Spring提供了一些关于Kafka的消息头：

  • KafkaHeaders.TOPIC
  • KafkaHeaders.PARTITION_ID
  • KafkaHeaders.MESSAGE_KEY
  • KafkaHeaders.TIMESTAMP
